# QueueFlow Project Documentation

![Project Status](https://img.shields.io/badge/status-active-success.svg)  

## 1. Escopo  
### 1.1 Organização Cliente - Restaurante IPEN  
Descrição da organização e suas expectativas em relação ao projeto.

### 1.2 Objetivo  
O objetivo deste projeto é desenvolver um sistema que otimize a fila de espera para atendimento no restaurante IPEN.

### 1.3 Público-Alvo  
O público-alvo do projeto são os visitantes do restaurante IPEN.

### 1.4 Matriz de Requisitos  
| Código   | Descrição                                                    |
|----------|--------------------------------------------------------------|
| REQ01    | O sistema deve permitir que o cliente faça sua reserva.      |
| REQ02    | O sistema deve enviar uma confirmação da reserva via e-mail. |
| REQ03    | O sistema deve exibir o tempo estimado de espera.            |
| REQ04    | O sistema deve permitir cancelamento da reserva.             |
| REQ05    | O sistema deve ter um controle de capacidade máxima.         |
| REQ06    | O sistema deve permitir que os funcionários visualizem a fila.
| REQ07    | O sistema deve enviar notificações aos clientes.             |
| REQ08    | O sistema deve armazenar dados históricos das reservas.      |
| REQ09    | O sistema deve ser acessível em dispositivos móveis.         |
| REQ10    | O sistema deve ter uma interface intuitiva.                 |
| REQ11    | O sistema deve garantir a segurança dos dados dos usuários.  |
| REQ12    | O sistema deve ser escalável.                               |
| REQ13    | O sistema deve suportar múltiplos usuários simultaneamente.  |

## 2. Requisitos Funcionais  
| Código   | Descrição                                      | Critério de Aceitação                     |
|----------|------------------------------------------------|-------------------------------------------|
| REQ01    | Permitir reserva de mesa                       | O cliente deve conseguir realizar a reserva sem erros. |
| REQ02    | Envio de confirmação por e-mail                | O cliente deve receber um e-mail dentro de 5 minutos. |
| REQ03    | Exibir tempo estimado de espera                | O tempo deve ser atualizado em tempo real. |
| REQ04    | Cancelamento de reservas                        | O cliente deve poder cancelar a reserva até 1 hora antes. |
| REQ05    | Controle de capacidade máxima                   | O sistema deve impedir reservas além da capacidade. |
| REQ06    | Visualização da fila por funcionários           | Funcionários devem visualizar reservas em tempo real. |
| REQ07    | Notificações aos clientes                       | Clientes devem ser notificados 10 minutos antes. |
| REQ08    | Armazenamento de dados históricos               | O sistema deve manter um registro de todos os atendimentos. |

## 3. Requisitos Não-Funcionais  
| Código   | Descrição                                      | Critério de Aceitação                     |
|----------|------------------------------------------------|-------------------------------------------|
| REQ09    | Acessibilidade móvel                            | O sistema deve ser responsivo em dispositivos móveis. |
| REQ10    | Interface intuitiva                            | O usuário deve ser capaz de navegar sem dificuldades. |
| REQ11    | Segurança dos dados                            | Os dados devem ser criptografados e seguros. |
| REQ12    | Escalabilidade                                 | O sistema deve suportar a adição de mais funcionalidades sem impacto. |
| REQ13    | Suporte a múltiplos usuários                   | O sistema deve permitir no mínimo 100 usuários simultâneos. |
